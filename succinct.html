Succinct | Presentation Generator

Developed using: HTML, CSS, PHP, Flickr Photosearch API, Google Natural Language Processing API

collaborators: Nishir Shelat, Teresa Liu, Sarah Lu, Teyva Chee
Created at Hack@UCI in Irvine, CA. Jan 2017.

Inspiration

Presentations are ubiquitous in modern education; both students and teachers are required on a regular basis to read through large amounts of text and make slides that summarizes this information. This type of reading and synthesizing information takes a lot of time, and is inefficient because presentations are not supposed to contain irrelevant or less important details. We wanted to help students and teachers out by giving them a tool to generate succinct, to-the-point slides based on any chunk of text.

Overview

Succinct generates images for each slide based on its title, and converts essays, paragraphs, and sentences to bullet format. Once the slide title and text are pasted into the box, a slide appears; at the top is the title. On the left, four relevant images appear (all of which can be copied and pasted to the student's choice of presentation software). On the right, bullet points with relevant words are shown; each bullet point corresponds with a sentence from the text, so the longer the text, the more bullets will be generated.

Our process

For the text generation, we used the Google Cloud Natural Language API to parse sentences into keywords and word types which were converted to a JSON file. We parsed the file so that the most important words would be printed out as bullets and displayed onto the slide.

Images were generated by using the Flickr Photosearch API to search for images by tag, which returned the JSON file containing the ID's of the photos. We used PHP to extract the ID's and stored them into an array, as well as to convert each ID into a URL. Using a foreach loop, we displayed 4 images which were resized and cropped using CSS to create a 2x2 grid of square photos.

Challenges

We ran into two main issues in this project. The first was finding an image-searching API that worked for our project. We initially implemented the Wikipedia API which returned a JSON file of the names of the images. However, since we found the URL associated with the image by concatenating the name of the image with a default URL, if the image was renamed or updated, the URL that we found would be invalid. We tried out multiple photosearch API's by Instagram, Tumblr, or Pinterest, but they either had a similar problem or required a certain fee. Thus we settled on using the Flickr API. The second issue we ran into was that the bullets did not make grammatical sense. We were able to extract the keywords as well as whether the word was a noun, adjective, etc. but we weren't able to concatenate these keywords into a bullet that would make grammatical sense. In the end, we simply separated these keywords using commas.

Reflections

This was my first hackathon project, and working on it really changed my outlook on what I could learn and accomplish in merely a few days. Furthermore, it was the first hackathon that over half of my team had attended, and everyone learned a new programming language.

Some additional features we were planning on adding include: a next button that keeps the original slides but refreshes the form to generate a new slide, exporting slides into a PDF that can be downloaded, creating a presentation view mode to make the presentation directly from the website, improving algorithm for parsing through outputted JSON file to generate grammatically correct bullets, and a bibliography generator.
